All canonicals use `https://commercialfinancereferrals.com/` with clean URLs (no `.html`).

### Issue: Redirect page canonical

| File | Line | Current canonical | Should be |
|------|------|-------------------|-----------|
| `commercial-lending-iso-agreement.html` | 12 | `https://commercialfinancereferrals.com/commercial-lending-iso-agreement` | `https://commercialfinancereferrals.com/commercial-lending-iso-program#agreement` |

**Impact:** Canonical points to the redirect page itself instead of the target. For a noindex redirect page, the canonical should point to the target so search engines consolidate signals correctly.

## 8. Sitemap

**File:** `sitemap.xml` (generated by `scripts/generate-sitemap.js`)

- **Build-time:** Sitemap is regenerated during `npm run build`. At that point `dist/` does not exist, so no `/dist/` URLs are included. Deployed sitemap is correct.
- **Committed file:** If `generate-sitemap.js` is run locally when `dist/` exists, it will include `/dist/` URLs. Those URLs 404 in production. Fix: exclude `dist` in `generate-sitemap.js` walk.

---

## Recommendations

1. **commercial-lending-iso-agreement.html:** Update canonical to `https://commercialfinancereferrals.com/commercial-lending-iso-program#agreement` so it points to the target page.
2. **generate-sitemap.js:** Add `dist` to the excluded directories in the walk function to avoid `/dist/` URLs if run when dist exists.
